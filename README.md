# Londoolink AI тАФ Your Intelligent Digital Twin

> Built for the **Auth0 AI Agents Hackathon 2026** & **Global Hack Week: API**

Londoolink is a multi-agent AI system with **persistent memory** that acts on your behalf across Gmail, Google Calendar, and Notion тАФ while keeping your credentials locked in **Auth0 Token Vault**. Your agents never hold raw tokens. You stay in control.

**Live Demo:** https://londoolink-ai.vercel.app  
**Backend API:** https://londoolink-abc635b5fe07.herokuapp.com  
**Try Demo Account:** Click "Try Demo" on the login page

---

## The Core Idea

Most AI assistants are passive and forgetful тАФ they wait for you to ask and forget everything after. Londoolink is different:

1. **Proactive Intelligence** тАФ Agents analyze your data and generate daily briefings automatically
2. **Persistent Memory** тАФ Powered by Backboard.io, the AI remembers your preferences across sessions
3. **Secure by Design** тАФ Auth0 Token Vault stores OAuth tokens, Auth0 FGA controls permissions
4. **Context-Aware** тАФ Thread-based conversations maintain context for follow-up questions
5. **Multi-Platform** тАФ Integrates Gmail, Google Calendar, Notion, and more

### How It Works

1. You connect Google and Notion via Auth0 OAuth
2. Auth0 Token Vault securely stores your access tokens (encrypted at rest)
3. LangGraph agents analyze your emails, calendar, and Notion pages
4. Backboard.io stores your preferences and conversation history
5. You get a personalized daily briefing based on your learned preferences
6. Urgent tasks trigger SMS alerts via Africa's Talking

Your agents act on your behalf. Auth0 ensures they only access what you've permitted. Backboard.io ensures they remember what matters to you.

## Architecture

![Architecture Diagram](docs/auth0-architecture.svg)

```
Frontend (Next.js 14)
    тЖУ Auth0 Universal Login + Step-Up Auth
Backend (FastAPI)
    тЖУ JWT Verification + Auth0 FGA Authorization
    тЖУ Auth0 Token Vault (OAuth token storage)
    тЖУ LangGraph Multi-Agent System
        тФЬтФАтФА Email Agent (Gmail readonly)
        тФЬтФАтФА Calendar Agent (Calendar readonly)
        тФЬтФАтФА Notion Agent (read_content)
        тФФтФАтФА Priority Agent тЖТ SMS Alert (Africa's Talking)
    тЖУ Backboard.io (AI Memory & RAG)
    тЖУ Neon PostgreSQL (user data, audit logs)
```

### Auth0 Integration
- **Universal Login** тАФ All auth flows through Auth0 (Google OAuth, email/password, MFA)
- **Token Vault** тАФ Google and Notion OAuth tokens stored in Auth0, never in our DB
- **Fine-Grained Authorization (FGA)** тАФ Permission-based access control for sensitive operations
- **Step-Up Authentication** тАФ Re-authentication required for high-security actions
- **M2M Client** тАФ Backend uses client credentials to retrieve tokens from vault
- **MFA** тАФ Multi-factor authentication delegated entirely to Auth0

### Backboard.io Integration
- **Persistent Memory** тАФ AI remembers user preferences across sessions
- **Cloud-Based RAG** тАФ Semantic search for emails, calendar events, and documents
- **Thread Management** тАФ Conversation history for follow-up questions
- **Automatic Learning** тАФ Agents learn from your interactions over time

### LangGraph Agents
- **Email Agent** тАФ Reads Gmail with `gmail.readonly` scope, surfaces urgent messages
- **Calendar Agent** тАФ Reads Google Calendar, flags conflicts and upcoming meetings
- **Notion Agent** тАФ Reads Notion workspace pages for action items
- **Priority Agent** тАФ Synthesizes all agent outputs into a personalized briefing
- **Data Collection Agent** тАФ Orchestrates data gathering from all integrated services

## Quick Start

### Prerequisites
- Python 3.12+
- Node.js 18+
- Auth0 account
- Neon PostgreSQL database

### Backend
```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

cp .env.example .env
# Fill in AUTH0_DOMAIN, AUTH0_CLIENT_ID, AUTH0_CLIENT_SECRET,
# AUTH0_M2M_CLIENT_ID, AUTH0_M2M_CLIENT_SECRET, DATABASE_URL, GEMINI_API_KEY

alembic upgrade head
uvicorn app.main:app --reload --port 8000
```

### Frontend
```bash
cd app-frontend
npm install

# .env.local
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
NEXT_PUBLIC_AUTH0_DOMAIN=your-tenant.auth0.com
NEXT_PUBLIC_AUTH0_CLIENT_ID=your-client-id
NEXT_PUBLIC_AUTH0_AUDIENCE=http://localhost:8000

npm run dev
```

## Auth0 Setup

### 1. Create Applications
- **Regular Web Application** тЖТ Note Client ID and Secret (for frontend)
- **Machine-to-Machine Application** тЖТ Authorize for your API (for backend Token Vault access)

### 2. Create API
- Create an API with identifier matching your backend URL
- Enable RBAC and add permissions if needed

### 3. Configure Social Connections
- Enable Google social connection with your Google OAuth credentials
- Enable Token Vault on the Google connection (Connected Accounts for Token Vault)
- Configure scopes: `gmail.readonly`, `calendar.readonly`

### 4. Set Up Fine-Grained Authorization (FGA)
- Create FGA store for permission management
- Define authorization model for sensitive operations
- Configure permission checks in your backend

### 5. Configure Callbacks
- Set Allowed Callback URLs: `https://your-frontend.vercel.app/auth/callback`
- Set Allowed Logout URLs: `https://your-frontend.vercel.app`
- Set Allowed Web Origins: `https://your-frontend.vercel.app`

### 6. Enable Step-Up Authentication
- Configure step-up authentication rules for sensitive operations
- Set ACR values for different security levels
